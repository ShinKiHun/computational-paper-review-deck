# Computational Paper Review Presentation Style Guide

This is the working standard for weekly paper-review materials.

## Core Format

- Use a vertical scrolling HTML page by default.
- Treat each section as one presentation beat, not as a dense document page.
- Use an assertion-evidence structure: the section headline states the claim, and the main figure or diagram provides the evidence.
- Keep the first screen focused on the paper's central tension and the authors' answer.
- Use figures at large scale. Do not bury paper figures inside small decorative cards.

## Story Flow

1. One-sentence thesis
2. Problem or anomaly
3. Old model vs new model
4. Key computational evidence
5. Key experimental evidence
6. Validation stack
7. Critical read
8. Discussion questions
9. Take-home message

## Design Rules

- Prefer claim-style headlines over topic labels.
- Use fewer bullets and more visual hierarchy.
- Avoid empty boxes. If a panel has only two short bullets, redesign it as a compact annotation or metric.
- Keep color functional: one color for mechanism, one for limitation, one for validation.
- Maintain consistent alignment and image sizing across sections.
- Make critical discussion explicit; do not end with only a summary.
